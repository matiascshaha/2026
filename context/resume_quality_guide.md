# Resume Quality Guide (Matias Perichon)

Derived from Deep Research synthesis provided on March 10, 2026.  
Purpose: produce interview-grade, ATS-safe, one-page resume and one-page cover letter for QA/SDET roles, with Apple-like role fit.

## Non-Negotiables
- Do not invent metrics, scope, leadership, or experience.
- Keep professional and personal project experience separate.
- Tone: concise, technical, specific, no corporate fluff.
- Resume target: 1 page.
- Cover letter target: 1 page.

## What Matters Most (Priority Order)
1. Clear evidence of bug detection, issue isolation, and fix verification loops.
2. Evidence of both structured testing and ad-hoc investigation capability.
3. Cross-functional communication and coordination signal.
4. Python automation credibility tied to quality outcomes.
5. Outcome-oriented bullets (not task lists), without fabricated numbers.
6. ATS-safe formatting and fast scan readability.
7. Relevant domain signal (spreadsheet/data/finance familiarity) when role requests it.

## Resume Structure (Default)
- Header: name, location, email, phone.
- Summary: 1-2 lines max, role-targeted.
- Experience: reverse chronological, role-fit bullets first.
- Education: compact.
- Skills: compact grouped index (proof stays in Experience).
- Projects: separate section, 1-2 bullets max for each included project.

## Content Budget
- Dexcom (current role): 4-6 bullets.
- Prior roles: 2-3 bullets each.
- Personal project: 1-2 bullets.
- Bullets should prioritize role-fit evidence before infrastructure depth.

## Bullet Standard
Each bullet should follow:
- Action verb + scope/system + method/tool + outcome.

Prefer bullets that include:
- issue isolation signal
- verification loop signal
- structured coverage signal
- communication/coordination signal

Avoid:
- vague responsibility phrasing
- long tool dumps
- generic claims with no evidence

## Role-Fit Rules (Apple-Like QA Targets)
- Foreground: bug isolation, bug report quality, verification discipline.
- Foreground: structured + ad-hoc testing capability.
- Foreground: communication across teams.
- Foreground: Python automation quality leverage.
- Include spreadsheet/data familiarity only if truthful and compact.
- De-emphasize infra scale details unless directly tied to QA outcomes.

## ATS and Formatting Rules
- Single-column layout.
- Standard headings (`Summary`, `Experience`, `Skills`, `Education`, `Projects`).
- No tables, no graphics, no text boxes, no critical content in headers/footers.
- Consistent date formatting (`Month YYYY`).
- Keep files text-based and lightweight.

## Cover Letter Rules
- 3-4 short paragraphs.
- Paragraph 1: role + direct fit statement.
- Paragraph 2: strongest professional proof mapped to core role needs.
- Paragraph 3: second proof mapped to preferred signals.
- Closing: concise interest + conversation invitation.
- Keep claims fully defensible and specific.

## Quality Gate Before Export
- No typos.
- No contradictions between resume and cover letter.
- Top third conveys fit in seconds.
- At least one explicit structured-testing signal.
- At least one explicit issue-isolation / verification signal.
- Personal project remains separate from professional experience.
